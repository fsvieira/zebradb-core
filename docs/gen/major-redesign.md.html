  • BeastDB is now being used for processing, which persists processing data on disk, allowing for resumption in case of a stop or crash. Additionally, this approach enables better memory handling since not all branches need to be in memory.
  • Domains have been added.
  • Hidden/body definitions and queries have been added.
  • A not-unify operator has been implemented.
